1. Update `backend/src/app.ts` to mount everything in correct order:

   **CRITICAL MIDDLEWARE ORDER (from RESEARCH.md anti-patterns):**
   ```
   1. trust proxy (already set in Plan 01)
   2. Health route (already mounted)
   3. Webhook middleware (MUST be before express.json() — consumes raw body for HMAC)
   4. CORS headers for widget (Access-Control-Allow-Origin: *)

   6. Report route with rate limiter and multer
   ```

   Updated `createApp()`:
   ```typescript
   import express from "express";
   import { healthRouter } from "./routes/health.js";
   import { reportRouter } from "./routes/report.js";
   import { reportLimiter } from "./middleware/rateLimit.js";
   import { webhookMiddleware } from "./middleware/webhook.js";

   export function createApp() {
     const app = express();

     // Trust Railway reverse proxy for correct client IP
     app.set("trust proxy", 1);

     // 1. Health check — always available
     app.use("/health", healthRouter);

     // 2. Webhook HMAC middleware — MUST be before express.json()
     //    createNodeMiddleware buffers raw body internally for HMAC verification
     //    Body stream can only be consumed once — if express.json() reads it first, HMAC fails
     app.use(webhookMiddleware);

     // 3. CORS for widget cross-origin POST
     app.use((req, res, next) => {
       res.header("Access-Control-Allow-Origin", "*");
       res.header("Access-Control-Allow-Methods", "GET, POST, OPTIONS");
       res.header("Access-Control-Allow-Headers", "Content-Type");
       if (req.method === "OPTIONS") {
         return res.sendStatus(204);
       }
       next();
     });

     // 4. JSON body parser for non-multipart routes (after webhook middleware)
     app.use(express.json());

     // 5. Report route with rate limiter (multer handles multipart internally)
     app.post("/report", reportLimiter, reportRouter);

     return app;
   }
   ```

   **Note on report route mounting:** The reportRouter internally uses `upload.array("screenshots", 10)` as route-level middleware, so multer only runs on the /report path. The rate limiter applies before multer.

   **Wait — Router mounting correction:** If reportRouter is a Router with `router.post("/", ...)`, then mounting as `app.post("/report", reportLimiter, reportRouter)` won't work correctly. Instead, either:
   - Option A: `app.use("/report", reportLimiter, reportRouter)` — router handles POST internally
   - Option B: Export the handler function directly, not a Router

   Use Option A: `app.use("/report", reportLimiter, reportRouter)` with reportRouter being a Router that has `router.post("/", upload.array(...), handler)`.
