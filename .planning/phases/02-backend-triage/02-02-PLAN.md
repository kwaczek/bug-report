<task type="auto">
  <name>Task 1: Implement ImgBB upload service and Anthropic triage service</name>
  <files>backend/src/services/imgbb.ts, backend/src/services/triage.ts</files>
  <action>
1. Create `backend/src/services/imgbb.ts`:

   **uploadToImgBB(buffer: Buffer, filename: string): Promise&lt;string&gt;**
   - Convert buffer to base64 string
   - POST to `https://api.imgbb.com/1/upload` with URLSearchParams: key (from `process.env.IMGBB_API_KEY`), image (base64), name (filename)
   - Return `data.data.url` from response
   - On failure: throw descriptive error with HTTP status

   **uploadScreenshots(files: Express.Multer.File[]): Promise&lt;string[]&gt;**
   - Map over files, call uploadToImgBB for each
   - Use `Promise.allSettled()` — not `Promise.all()` — so one failure doesn't block others
   - For fulfilled: collect URL
   - For rejected: log warning `[imgbb] upload failed for ${filename}: ${reason}`, push placeholder string `"[screenshot unavailable]"` instead
   - Return array of URLs/placeholders (graceful degradation per RESEARCH.md Pitfall 3)

2. Create `backend/src/services/triage.ts`:

   **triageReport(report: { subject: string; description: string; metadata: string }): Promise&lt;TriageResult&gt;**
   - Import `Anthropic` from `@anthropic-ai/sdk`
   - Import `zodOutputFormat` from `@anthropic-ai/sdk/helpers/zod`
   - Import `z` from `zod`
   - Define `TriageSchema` using zod:
     ```typescript
     const TriageSchema = z.object({
       verdict: z.enum(["auto-fix", "review", "spam"]),
       confidence: z.number().min(0).max(1),
       reasoning: z.string(),
     });
     ```
   - Create Anthropic client (reads ANTHROPIC_API_KEY from env automatically)
   - Call `client.messages.parse()` with:
     - model: "claude-sonnet-4-5"
     - max_tokens: 512
     - system prompt: "You are a bug report triage classifier. Classify bug reports into three categories based on confidence: auto-fix (confidence > 0.8, clear reproducible bug with specific steps), review (confidence 0.4-0.8, ambiguous or needs human judgment), spam (confidence < 0.4, nonsense, test data, advertising, or not a bug report). Return your verdict, confidence score, and brief reasoning."
     - user message: formatted with Subject, Description, Browser metadata
     - output_config: `{ format: zodOutputFormat(TriageSchema) }`
   - Return `message.parsed_output!`
   - **CRITICAL: Wrap entire function in try/catch.** On ANY error (API failure, parse failure, null parsed_output):
     - Log warning: `[triage] AI triage failed, defaulting to review: ${error}`
     - Return fallback: `{ verdict: "review", confidence: 0.5, reasoning: "Triage unavailable — defaulting to manual review" }`
   - This ensures triage failure never blocks report submission (RESEARCH.md Pitfall 4)
  </action>

  </verify>
  <done>imgbb.ts exports uploadToImgBB and uploadScreenshots with graceful degradation; triage.ts exports triageReport with structured Anthropic output and safe fallback to "review" on any failure</done>

1. Create `backend/src/services/github.ts`:

   **Create Octokit instance:**
   ```typescript
   import { Octokit } from "@octokit/rest";
   const octokit = new Octokit({ auth: process.env.GITHUB_TOKEN });
   ```

   **buildIssueBody(args): string**
   - Accept: `{ description: string; screenshotUrls: string[]; metadata: BugMetadata; triageResult: TriageResult }`
   - Generate markdown body matching the template from RESEARCH.md:
     - `## Bug Report` section with description
     - `## Screenshots` section: map URLs to `![Screenshot N](url)`, or "_No screenshots attached_" if empty
     - `## Environment` table: URL, Browser (userAgent), Screen (widthxheight), Language, Timestamp
     - `## Triage` table: Verdict, Confidence (2 decimal places), Reasoning
   - Return the markdown string

   **createGitHubIssue(args): Promise&lt;{ issueId: number; issueUrl: string }&gt;**
   - Accept: `{ owner: string; repo: string; subject: string; description: string; screenshotUrls: string[]; metadata: BugMetadata; triageResult: TriageResult }`
   - Determine label based on verdict: `triageResult.verdict === "auto-fix" ? "auto-fix" : "needs-review"`
   - Call `octokit.rest.issues.create()` with:
     - owner, repo from args
     - title: subject
     - body: `buildIssueBody({ description, screenshotUrls, metadata, triageResult })`
     - labels: `["bug-report", verdictLabel]`
   - Return `{ issueId: data.number, issueUrl: data.html_url }`

   **ensureLabelsExist(owner: string, repo: string): Promise&lt;void&gt;**
   - Define required labels: `[{ name: "bug-report", color: "d73a4a" }, { name: "auto-fix", color: "0e8a16" }, { name: "needs-review", color: "fbca04" }]`
   - For each label, call `octokit.rest.issues.createLabel({ owner, repo, name, color })`
   - Catch errors where `status === 422` (label already exists) — silently continue
   - Log: `[github] labels ensured for ${owner}/${repo}`
   - This is idempotent — safe to call at every startup (RESEARCH.md Pitfall 6)
