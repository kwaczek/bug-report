<objective>
Create the relay server package — a minimal Express service that receives auto-fix payloads from Railway, deduplicates them, queues per-project writes, and produces fix_plan.md files for Ralph to consume.

Purpose: The relay is the bridge between the cloud-hosted Railway backend and the local Ralph workspace. It receives authenticated POSTs, validates payloads, prevents duplicate processing from webhook retries, serializes writes per project, and writes fix_plan.md in Ralph's verified format. Ralph's polling loop picks up the file within ~5 seconds.

Output: Complete `relay/` package with Express server, auth middleware, dedup service, per-project queue, fix_plan.md writer, and POST /fix endpoint.
</objective>

Ralph fix_plan.md format (verified from /Users/miro/.ralph/ralph_loop.sh):
```markdown
# Bug Fix: <issue title>

**Issue:** <GitHub issue URL>
**Reported:** <timestamp>
**Repo:** <owner/repo>

## Investigation
- [ ] Read the bug report at <issue URL> — understand the exact failure, reproduce steps, and identify affected code paths

## Implementation
- [ ] Identify the root cause in the codebase
- [ ] Implement the minimal fix — do not refactor unrelated code
- [ ] Run the test suite to verify no regressions

## Delivery
- [ ] Commit the fix with message: "fix: <brief description> (closes #<issueId>)"
- [ ] Push to main branch — Railway auto-deploys on push
```

Ralph behavior facts:
- Reads fix_plan.md at START of each loop after 5-second sleep (polling, not file-watching)
- Path: `<project_dir>/.ralph/fix_plan.md`
- Completion: exits when total > 0 AND completed == total (all `[x]`)
- Does NOT require fix_plan.md to exist at startup

  app.use((req, res, next) => {

4. Define Zod schema `FixRequestSchema` for payload validation:
   ```typescript
   const FixRequestSchema = z.object({
     issueId: z.number(),
     issueUrl: z.string().url(),
     issueTitle: z.string().min(1),
     owner: z.string().min(1),
     repo: z.string().min(1),
     triageResult: z.object({
       verdict: z.literal('auto-fix'),
       confidence: z.number().min(0).max(1),
       reasoning: z.string(),
     }),
     reportData: z.object({
       subject: z.string(),
       description: z.string(),
       screenshotUrls: z.array(z.string()),
     }),
   });
   ```

5. Export `fixRouter = Router()`

6. Implement `fixRouter.post('/', async (req, res) => { ... })`:
   a. Validate payload with `FixRequestSchema.safeParse(req.body)` — return 400 on failure with error details
   b. Build dedup key: `${data.owner}/${data.repo}#${data.issueId}`
   c. Check `isDuplicate(dedupeKey)` — if duplicate, log and return `200 { status: 'duplicate', key }`
      (Return 200, NOT 4xx — error codes trigger more retries from GitHub/Railway)
   d. Call `markSeen(dedupeKey)` — persist to file
   e. Log triage decision via `appendTriageLog({ timestamp, issueId, owner, repo, verdict, confidence, reasoning })`
   f. Enqueue the fix_plan.md write job:
      ```typescript
      enqueue(data.repo, async () => {
        // Check if Ralph is busy with an existing job (Pitfall 1 from research)
        const busy = await isProjectBusy(data.repo);
        if (busy) {
          console.warn(`[fix] project ${data.repo} has in-progress fix_plan.md — queuing delayed retry`);
          // Re-enqueue with delay: wait 60s then try again
          await new Promise(resolve => setTimeout(resolve, 60_000));
          const stillBusy = await isProjectBusy(data.repo);
          if (stillBusy) {
            console.error(`[fix] project ${data.repo} still busy after retry — skipping ${dedupeKey}`);
            return;
          }
        }
        const content = buildFixPlan(data);
        await writeFixPlan(data.repo, content);
      });
      ```
   g. Return `202 { status: 'accepted', key: dedupeKey }` immediately — do NOT await the queue
