Create `backend/src/services/relay.ts`:

1. Define `RelayFixPayload` interface (the contract between backend and relay):
   ```typescript
   export interface RelayFixPayload {
     issueId: number;
     issueUrl: string;
     issueTitle: string;
     owner: string;
     repo: string;
     triageResult: {
       verdict: 'auto-fix';
       confidence: number;
       reasoning: string;
     };
     reportData: {
       subject: string;
       description: string;
       screenshotUrls: string[];
     };
   }
   ```

2. Export `notifyRelay(payload: RelayFixPayload): Promise<void>`:
   - Read `RELAY_URL` from `process.env.RELAY_URL` (e.g. `https://relay.yourdomain.com/fix`)
   - If `RELAY_URL` is not set, log `[relay] RELAY_URL not set — skipping relay notification` and return (do NOT throw)
   - POST to RELAY_URL with:
     - `Content-Type: application/json`
     - `Authorization: Bearer ${process.env.RELAY_SECRET}`
     - Body: JSON.stringify(payload)
   - Use native `fetch()` (Node 18+ built-in, no extra dependency)
   - If response is ok (2xx), return immediately — delivery succeeded
   - If response is NOT ok or fetch throws (network error): start background retry loop (do NOT await — fire and forget)
   - **Background retry with relabel on exhaustion** (per user decision: "On delivery failure after all retries: relabel GitHub issue from `auto-fix` to `needs-review`"):
     ```typescript
     const RETRY_DELAYS_MS = [1*60_000, 5*60_000, 15*60_000, 30*60_000, 60*60_000]; // 1, 5, 15, 30, 60 min

     async function retryRelayDelivery(payload: RelayFixPayload): Promise<void> {
       for (const delay of RETRY_DELAYS_MS) {
         console.log(`[relay] retrying in ${delay/60_000}min for ${payload.owner}/${payload.repo}#${payload.issueId}`);
         await new Promise(r => setTimeout(r, delay));
         try {
           const res = await fetch(process.env.RELAY_URL!, { /* same headers/body */ });
           if (res.ok) {
             console.log(`[relay] retry succeeded for ${payload.owner}/${payload.repo}#${payload.issueId}`);
             return; // Success — exit retry loop
           }
         } catch { /* continue to next retry */ }
       }
       // All retries exhausted — relabel issue
       console.error(`[relay] all retries exhausted for ${payload.owner}/${payload.repo}#${payload.issueId} — relabeling to needs-review`);
       await relabelIssue(payload.owner, payload.repo, payload.issueId, 'auto-fix', 'needs-review');
     }
     ```
   - The initial `notifyRelay()` call catches the first failure, spawns `retryRelayDelivery()` as a detached promise (with `.catch()` error handler), and returns immediately — relay failure must not crash the report pipeline
   - Import `relabelIssue` from `./issue-relabel.js`
